    calculate-aspect-ratio

    @function calculate-aspect-ratio($ratio: false, $x: 1, $y: 1, $unit: "%", $decimal-accuracy: 2) { ... }

    Description

    Get the calculated relative vertical aspect ratio for given values, either as a SASS list, map or string, the latter of which is sought in the $aspect-ratios variable.

    Parameters

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $ratio

    The complete ratio. Use either this argument OR the next two.

    		List or Map or Stringfalse
    $x

    The x (horizontal) value of the desired ratio

    		Number1
    $y

    The y (vertical) value of the desired ratio

    		Number1
    $unit

    The default CSS unit to be used. Most likely, if not the default % then maybe vw.

    		String"%"
    $decimal-accuracy

    The number of decimal places to which the result should be rounded.

    		Number2

    Returns

    Number —

    A number representing the vertical ratio with respect to the passed arguments

    Example

    SCSS usage - Keyword aspect ratio from $aspect-ratios Map

    .selector {
	padding-bottom: calculate-aspect-ratio("widescreen");
}

    CSS output

    .selector {
	padding-bottom: 41.84%;
}

    SCSS usage - Custom ratio passed as a List

    .selector {
	min-height: calculate-aspect-ratio(16 9, $unit: vw);
}

    CSS output

    .selector {
	min-height: 56.25vw;
}

    Throws

    • Warning - If no valid aspect ratio could be acertained by the given parameters. Only a warning, a result of 100[unit] will still be returned.

    Requires

    • [function] is-list
    • [function] is-map
    • [function] is-string
    • [function] is-number

    • [function] decimal-round

    • [variable] aspect-ratios

    aspect-ratio-box

    @mixin aspect-ratio-box($pseudo: false, $ratio: false, $x: 1, $y: 1, $unit: "%", $decimal-accuracy: 2) { ... }

    Description

    Create a box element with a fixed aspect ratio via the css padding-hack.

    Parameters

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $pseudo

    Should the box be generated via a pseudo element. Set this to true when flex-basis alone defines the parent width.

    		Boolfalse
    $ratio

    The complete ratio. Use either this argument OR the next two.

    		List or Map or Stringfalse
    $x

    The x (horizontal) value of the desired ratio

    		Number1
    $y

    The y (vertical) value of the desired ratio

    		Number1
    $unit

    The default CSS unit to be used. Most likely, if not the default % then maybe vw.

    		String"%"
    $decimal-accuracy

    The number of decimal places to which the result should be rounded.

    		Number2

    Content

    This mixin allows extra content to be passed (through the @content directive).

    Example

    SCSS usage - Create a pseudo element with the passed keyword aspect ratio from $aspect-ratios Map

    .selector {
	@include aspect-ratio-box(true, "4x3");
}

    CSS output

    .selector::before {
	content: "";
	display: block;
	padding-top: 75%;
	position: relative;
	width: 100%;
}

    SCSS usage - Custom ratio passed as individual x and y and without creating a pseudo element.

    .selector {
	@include aspect-ratio-box($x: 42, $y: 24, $unit: vw);
}

    CSS output

    .selector {
	padding-top: 57.14vw;
	position: relative;
}

    Output

    Padding-top value to create the desired aspect ratio, either nested directly under the parent selector or, optionally, as a before pseudo element.

    Requires

    • [function] calculate-aspect-ratio

    mq

    @mixin mq($min-width: false, $max-width: false, $breakpoint-map: $breakpoints) { ... }

    Description

    Super primitive mixin for min-width and/or max-width media queries.

    Parameters

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $min-width— noneNumber or Stringfalse
    $max-width— noneNumber or Stringfalse
    $breakpoint-map— noneMap$breakpoints

    Content

    This mixin allows extra content to be passed (through the @content directive).

    Example

    SCSS usage

    @include mq(sm) {
	font-size: 1rem;
}

    CSS output

    @media mq(min-width: 576px) {
	font-size: 1rem;
}

    SCSS usage

    @include mq(md, lg) {
	font-size: 1rem;
}

    CSS output

    @include mq(min-width: 768px, max-width: calc(960px - 1px)) {
	font-size: 1rem;
}

    SCSS usage - custom breakpoint

    @include mq($max-width: 100vh) {
	font-size: 1rem;
}

    CSS output - custom breakpoint

    @media mq(max-width: 100vh) {
	font-size: 1rem;
}

    Output

    • A media query around the @content

    Throws

    • No value found for #{ $min-width }

    • No value found for #{ $max-width }

    Requires

    • [function] is-string

    Description

    Create a font face rule. For complete documentation follow the link. Format Support: https://socialcompare.com/en/comparison/browser-fonts-support-comparison http://transfonter.org/formats

    Parameters

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $name

    Name of font-family

    		String— none
    $path

    Path to font files

    		String— none
    $weight

    Associated font-weight. Defaults to normal

    		String or Numbernull
    $style

    Associated font-style. Defaults to normal

    		Stylenull
    $exts

    Available formats

    		Listwoff2 woff

    Example

    SCSS usage

	font-style: normal;
	font-weight: 500;
